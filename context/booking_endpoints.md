# مرجع واجهات برمجة التطبيقات للحجوزات (Bookings API Reference) - CarHero

يحتوي هذا الملف على توثيق شامل لجميع الـ Endpoints الخاصة بنظام الحجوزات (الفورية والمجدولة) في CarHero، مع توضيح صلاحيات الوصول لكل مسار.

- **الرابط الأساسي (Base URL)**: `/api/v1/bookings`
- **نظام الحماية**: جميع المسارات تتطلب `JWT Token` وتخضع لنظام التحقق من الملكية (Ownership Verification) والأدوار (Roles).

---

## 1. حجوزات العملاء (Customer Operations)
**المسار الأساسي**: `/bookings`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/bookings` | `POST` | إنشاء حجز فوري (بنش، بطارية، إلخ). | العميل (User) |
| `/bookings/schedule` | `POST` | إنشاء حجز مجدول لموعد لاحق. | العميل (User) |
| `/bookings/my` | `GET` | قائمة حجوزاتي الفورية (يدعم الكاش). | العميل (صاحب الحساب) |
| `/bookings/schedule/my` | `GET` | قائمة حجوزاتي المجدولة (يدعم الكاش). | العميل (صاحب الحساب) |
| `/bookings/:id` | `GET` | تفاصيل حجز معين بالـ ID. | العميل / المزود / الأدمن |
| `/bookings/:id/cancel` | `PATCH` | إلغاء حجز فوري مع ذكر السبب. | العميل (قبل وصول الفني) |
| `/bookings/schedule/:id` | `DELETE` | حذف/إلغاء حجز مجدول. | العميل (User) |
| `/bookings/:id/review` | `POST` | تقييم الخدمة وإضافة تعليق. | العميل (بعد الاكتمال) |
| `/bookings/:id/track` | `GET` | تتبع موقع الفني المسند للحجز. | العميل صاحب الحجز |
| `/bookings/:id/pay` | `POST` | دفع تكلفة الحجز من المحفظة. | العميل (User) |

---

## 2. إدارة المزودين (Provider Operations)
**المسار الأساسي**: `/provider/bookings`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/provider/bookings/nearby` | `GET` | البحث عن الحجوزات القريبة (Gps). | المزود (Provider) |
| `/provider/bookings/:id/accept` | `PATCH` | قبول طلب الحجز المتاح. | المزود (Provider) |
| `/provider/bookings/:id/reject` | `PATCH` | رفض طلب الحجز (للمزودين المسندين). | المزود (Provider) |
| `/provider/bookings/:id/start` | `PATCH` | البدء في تنفيذ الخدمة (On the way). | المزود المعني بالحجز |
| `/provider/bookings/:id/complete` | `PATCH` | إنهاء الخدمة (تتحول الأرباح للمحفظة). | المزود المعني بالحجز |
| `/provider/bookings/history` | `GET` | سجل الحجوزات التي نفذها المزود. | المزود (Provider) |

---

## 3. لوحة الإدارة (Admin Operations)
**المسار الأساسي**: `/admin/bookings`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/admin/bookings` | `GET` | استعراض كافة الحجوزات في النظام. | الأدمن (Admin) |
| `/admin/bookings/stats` | `GET` | إحصائيات الحجوزات (عدد، نسب، دخل). | الأدمن (Admin) |
| `/admin/bookings/:id/status` | `PATCH` | تعديل حالة أي حجز يدوياً. | الأدمن (Admin) |

---

## 4. المميزات التقنية المدمجة (Internal Features)
*   **تحويل الأرباح الفوري**: عند الـ `complete` يتم تحويل 90% للمزود و 10% للمنصة آلياً.
*   **التشغيل اللحظي (WebSockets)**: يتم بث تحديثات الحالة لحظياً لتطبيقات الأجهزة المحمولة.
*   **نظام الـ Cron**: إلغاء تلقائي للحجوزات التي لم تُقبل خلال ساعتين.
*   **دقة البيانات**: استخدام MongoDB Geospatial Index للحصول على نتائج بحث دقيقة وسريعة.

---

> **تنبيه**: جميع العمليات المالية والمكانية تخضع لتدقيق صارم؛ أي تلاعب في الـ `providerId` أو الموقع الجغرافي يتم رفضه برمجياً لضمان مصداقية الخدمة.
