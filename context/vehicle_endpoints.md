# مرجع واجهات برمجة التطبيقات للمركبات (Vehicles API Reference) - CarHero

يحتوي هذا الملف على توثيق شامل لجميع الـ Endpoints الخاصة بنظام إدارة المركبات، الصيانة، والتذكيرات في CarHero.

- **الرابط الأساسي (Base URL)**: `/api/v1`
- **نظام الحماية**: جميع المسارات تتطلب `JWT Token`. يتم التحقق من الملكية (Ownership) تلقائياً لكل عملية.

---

## 1. إدارة المركبات (Core Vehicle Management)
**المسار الأساسي**: `/vehicles`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/vehicles` | `POST` | إضافة مركبة جديدة للمستخدم الحالي. | المستخدم (User) |
| `/vehicles/my` | `GET` | جلب جميع مركبات المستخدم الحالي (مع Pagination). | المستخدم |
| `/vehicles/search` | `GET` | البحث في مركبات المستخدم (الماركة، الموديل، اللوحة). | المستخدم |
| `/vehicles/:id` | `GET` | جلب تفاصيل مركبة معينة بالـ ID. | صاحب المركبة / الأدمن |
| `/vehicles/:id` | `PATCH` | تحديث بيانات مركبة (اللون، سنة الصنع، إلخ). | صاحب المركبة |
| `/vehicles/:id` | `DELETE` | حذف مركبة من النظام. | صاحب المركبة |
| `/vehicles/:id/set-default` | `PATCH` | تعيين المركبة كمركبة افتراضية للحساب. | صاحب المركبة |

## 4. واجهات الإدارة (Admin Endpoints)
**المسار الأساسي**: `/admin/vehicles`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/admin/vehicles` | `GET` | جلب جميع المركبات في النظام بالكامل. | الأدمن (Admin) |
| `/admin/vehicles/stats` | `GET` | إحصائيات عامة عن المركبات والماركات. | الأدمن |
| `/admin/vehicles/top-models` | `GET` | قائمة بأكثر موديلات السيارات انتشاراً. | الأدمن |
| `/admin/vehicles/distribution` | `GET` | توزيع المركبات كنسب مئوية حسب الماركة. | الأدمن |
| `/admin/vehicles/year-stats` | `GET` | إحصائيات المركبات حسب سنة الصنع. | الأدمن |
| `/admin/vehicles/user/:userId` | `GET` | جلب جميع مركبات مستخدم معين. | الأدمن |
| `/admin/vehicles/:id` | `DELETE` | حذف أي مركبة من النظام (إجراء إداري). | الأدمن |

---

## 5. منطق العمل (Business Logic)
*   **Default Vehicle Core**: لا يمكن للمستخدم حذف المركبة الافتراضية إلا إذا قام بتعيين مركبة أخرى كافتراضية أولاً.
*   **Smart Triggers**: التذكيرات تدعم خاصية "التكرار الذكي" (Recurring) لضمان تنبيه المستخدم دورياً.
*   **Data Validation**: يتم التحقق من صحة رقم الـ VIN وسنة الصنع برمجياً قبل الحفظ.

---

> **تحذير**: جميع العمليات تخضع للتحقق من الصلاحيات (Authorization). الوصول لبيانات مركبات الغير يؤدي لاستجابة `403 Forbidden`.
