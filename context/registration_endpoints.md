# مرجع واجهات برمجة التطبيقات للتسجيل والاعتماد (Registration & Approval API) - CarHero

يحتوي هذا المستند على تفاصيل نقاط الاتصال الخاصة بتسجيل مزودي الخدمة الجدد وعمليات المراجعة والاعتماد من قبل الإدارة.

- **الرابط الأساسي (Base URL)**: `/api/v1`
- **الحماية**: عمليات الاعتماد تتطلب `JWT Token` بصلاحيات `ADMIN`.

---

## 1. عمليات التسجيل (Signup Flow)

### أ. تسجيل مزود جديد (Provider Signup)
- **المسار**: `POST /auth/register`
- **الوصف**: يبدأ عملية التسجيل بإرسال البيانات الأساسية. إذا كان `accountType` هو `provider` فسيتم إنشاؤه بحالة معلقة.
- **هيكل البيانات (Request Body)**:
```json
{
  "phoneNumber": "+96650...",
  "password": "...",
  "fullName": "اسم المركز أو المزود",
  "accountType": "provider", // ضروري لتفعيل دورة حياة الاعتماد
  "isTermsAccepted": true
}
```

### ب. التحقق وتفعيل الطلب (OTP Verification)
- **المسار**: `POST /auth/verify-otp`
- **الوصف**: بعد التأكد من الرمز، يتم إنشاء ملف المزود رسمياً ولكنه يبقى `PENDING`.

---

## 2. عمليات الإدارة (Admin Review)

### أ. عرض الطلبات المعلقة (List Pending)
- **المسار**: `GET /admin/providers?status=pending`
- **الوصف**: جلب كافة المزودين الذين ينتظرون المراجعة والاعتماد.
- **Query Params**: `page`, `limit`.

### ب. الموافقة والاعتماد (Approve Provider)
- **المسار**: `PATCH /admin/providers/:id/approve`
- **الوصف**: اعتماد المزود وتفعيل حسابه ليتمكن من الدخول والظهور في التطبيق.
- **التأثير**: 
  - `registrationStatus` -> `approved`
  - `User.isActive` -> `true`
  - إرسال إشعار تهنئة للمزود.

### ج. رفض الطلب (Reject Provider)
- **المسار**: `PATCH /admin/providers/:id/reject`
- **الوصف**: رفض طلب التسجيل مع توضيح السبب.
- **هيكل البيانات (Body)**:
```json
{
  "reason": "المستندات المرفوعة غير واضحة أو منتهية الصلاحية"
}
```

---

## 3. قواعد العمل (Business Rules)

- **منع الدخول**: أي مستخدم بنوع `provider` لا يمكنه تجاوز بوابة تسجيل الدخول (`login`) إذا كانت حالته `pending` أو `rejected`. ستظهر له رسالة: "الحساب تحت المراجعة" أو "تم رفض الحساب".
- **عمر الطلب المعلق**: تظهر الطلبات المعلقة للأدمن مرتبة من الأقدم للأحدث لضمان سرعة المعالجة.
- **المستندات**: يُنصح المزود برفع مستنداته الثبوتية فور التسجيل ليتم تقييمه من قبل الإدارة بشكل أسرع.

---

> [!TIP]
> جميع إشعارات نظام التسجيل مؤتمتة وتصل عبر الـ WebSockets والـ Push Notifications لضمان تجربة سريعة للمزود والإدارة.
