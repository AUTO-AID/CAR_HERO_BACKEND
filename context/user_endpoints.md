# مرجع واجهات برمجة التطبيقات للمستخدمين (Users API Reference) - CarHero

يحتوي هذا الملف على توثيق شامل لجميع الـ Endpoints الخاصة بنظام إدارة المستخدمين، مع توضيح صلاحيات الوصول ومنطق العمل.

- **الرابط الأساسي (Base URL)**: `/api/v1`
- **نظام الحماية**: جميع المسارات تتطلب `JWT Token`. يتم التحقق من الملكية (Ownership) برمجياً لضمان الخصوصية.

---

## 1. عمليات المستخدم (User Operations)
**المسار الأساسي**: `/users`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/users/me` | `GET` | جلب الملف الشخصي للمستخدم الحالي. | المستخدم الحالي |
| `/users/me` | `PATCH` | تحديث بيانات المستخدم (الاسم، اللغة، الإشعارات). | المستخدم الحالي |
| `/users/me` | `DELETE` | إغلاق حساب المستخدم نهائياً (Soft Delete). | المستخدم الحالي |
| `/users/me/stats` | `GET` | جلب إحصائيات المستخدم (عدد الطلبات، النقاط، إلخ). | المستخدم الحالي |

---

## 2. واجهات الإدارة (Admin Operations)
**المسار الأساسي**: `/admin/users`

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/admin/users` | `GET` | جلب قائمة بجميع المستخدمين (مع Pagination). | الأدمن (Admin) |
| `/admin/users/:id` | `GET` | عرض تفاصيل مستخدم معين بالـ ID. | الأدمن |
| `/admin/users/:id` | `PATCH` | تحديث بيانات أي مستخدم في النظام. | الأدمن |
| `/admin/users/:id` | `DELETE` | تعطيل حساب مستخدم (إجراء إداري). | الأدمن |

---

## 3. منطق العمل والحماية (Backend Logic)
*   **Security (JWT)**: يجب إرسال الـ Token في الـ Header كـ `Bearer Token`.
*   **Ownership Protection**: أي محاولة للوصول لبيانات مستخدم آخر ستؤدي إلى خطأ `403 Forbidden`.
*   **Data Integrity**: لا يتم حذف بيانات المستخدم فزيائياً، بل يتم تحويل حالة `isActive` إلى `false` للحفاظ على السجلات المالية والطلبات المرتبطة به.
*   **Field Sanitization**: كلمة المرور وحقول الأمان محجوبة دائماً ولا تظهر في نتائج أي Endpoint.

---

> **ملاحظة**: لإجراء عمليات تسجيل الدخول أو استعادة كلمة المرور، يرجى مراجعة موديول **Auth**.
