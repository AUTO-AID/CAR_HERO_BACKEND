# مرجع واجهات برمجة التطبيقات للمحادثة (Chat API & Socket Reference) - CarHero

يحتوي هذا الملف على توثيق شامل لكافة الواجهات (REST) والأحداث اللحظية (WebSockets) لموديول المحادثة في CarHero.

- **الواجهة الأساسية (REST)**: `/api/v1/chat`
- **مسار الـ WebSocket**: `namespace: /chat`
- **نظام الحماية**: يتطلب `JWT Token` في الهيدر أو الـ Handshake.

---

## 1. واجهات الـ REST (HTTP REST API)

| المسار (Endpoint) | الطريقة (Method) | الوصف | صلاحيات الوصول |
| :--- | :--- | :--- | :--- |
| `/chat/conversations` | `POST` | إنشاء أو فتح محادثة (1-on-1) بين مستخدمين. | العميل / المزود |
| `/chat/conversations` | `GET` | جلب قائمة كافة المحادثات الخاصة بالمستخدم الحالي. | أي مستخدم مسجل |
| `/chat/:chatId/messages` | `GET` | جلب سجل الرسائل لمحادثة معينة (Paginated). | المشاركون في المحادثة |
| `/chat/upload` | `POST` | رفع ملف/صورة لاستخدامها في المحادثة (Max 5MB). | أي مستخدم مسجل |

---

## 2. أحداث الـ WebSockets (Socket.IO Events)

### ا- أحداث من العميل إلى الخادم (Client → Server)
| الحدث (Event) | البيانات (Payload) | الوصف |
| :--- | :--- | :--- |
| `join_chat` | `{ chatId: string }` | الانضمام لغرفة محادثة محددة لبدء الاستقبال. |
| `send_message` | `{ chatId, message, type, fileUrl? }` | إرسال رسالة جديدة (نص، صورة، موقع). |
| `typing` | `{ chatId, isTyping: boolean }` | إرسال حالة "جاري الكتابة" للطرف الآخر. |
| `message_read` | `{ chatId }` | إبلاغ الخادم والطرف الآخر بأن الرسائل تمت قراءتها. |
| `leave_chat` | `{ chatId }` | مغادرة غرفة المحادثة. |

### ب- أحداث من الخادم إلى العميل (Server → Client)
| الحدث (Event) | البيانات (Payload) | الوصف |
| :--- | :--- | :--- |
| `new_message` | كائن الرسالة كاملاً | وصول رسالة جديدة في المحادثة المنضم إليها. |
| `user_typing` | `{ userId, isTyping }` | إشعار العميل بأن الطرف الآخر يقوم بالكتابة حالياً. |
| `messages_marked_read` | `{ userId }` | إشعار العميل بأن الطرف الآخر قرأ رسائله. |
| `user_online` / `user_offline` | `{ userId }` | تحديث حالة اتصال الطرف الآخر. |

---

## 3. منطق الأمان والتحقق (Backend Security)
- **Room Membership**: يتم التحقق من عضوية المستخدم في المحادثة في قاعدة البيانات قبل السماح له بالانضمام للغرفة (`join_chat`).
- **Participant Validation**: لا يمكن فتح محادثة مرتبطة بطلب (`orderId`) إلا إذا كان الطرفان هما العميل والمزود الفعلي للطلب.
- **REST Protection**: جميع مسارات جلب التاريخ والرفع محمية بـ Guard يتحقق من ملكية المحادثة.

---

## 4. مرفقات المحادثة (Attachments)
يتم رفع الملفات عبر REST API لاستقرار الاتصال، ثم يتم بث رابط الملف (`fileUrl`) عبر الـ WebSocket كجزء من الرسالة.
