# مرجع واجهات برمجة التطبيقات للأدمن (Admin API Reference) - CarHero

يحتوي هذا الملف على توثيق شامل لكل الـ Endpoints المخصصة حصرياً للأدمن في نظام CarHero.

- **الرابط الأساسي (Base URL)**: `api/v1`
- **نظام الحماية**: جميع هذه المسارات تتطلب `JWT Token` بصلاحية `Role.ADMIN` (ما عدا الدخول وتجديد التوكن).

---

## 1. مجموعة الأدمن (Admin Module)
**المسار الأساسي**: `/admin`

### أ. المصادقة والحساب (Auth & Identity)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/login` | `POST` | تسجيل دخول الأدمن، يعيد Access Token و Refresh Token. |
| `/refresh-token` | `POST` | تجديد التوكنات المنتهية باستخدام الـ Refresh Token. |
| `/logout` | `POST` | تسجيل الخروج وإبطال الـ Refresh Token في قاعدة البيانات. |
| `/me` | `GET` | جلب بيانات الـ Profile للأدمن الحالي المسجل دخوله. |

### ب. إدارة المستخدمين (Users Management)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/users` | `GET` | جلب قائمة بجميع المستخدمين المسجلين (مع pagination). |
| `/users/search` | `GET` | البحث عن مستخدم معين باستخدام الاسم أو رقم الهاتف. |
| `/users/:id` | `GET` | جلب كافة تفاصيل ملف مستخدم معين. |
| `/users/:id/status` | `PATCH` | تفعيل أو تعطيل حساب مستخدم (حظر/فك حظر). |
| `/users/:id` | `DELETE` | حذف حساب مستخدم نهائياً من النظام. |

### ج. إدارة المزودين (Providers Management)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/providers` | `GET` | عرض قائمة المزودين (يمكن الفلترة بحالات مثل: Pending, Approved). |
| `/providers/:id` | `GET` | جلب تفاصيل المزود، أوراقه الثبوتية، وقائمة خدماته. |
| `/providers/:id/approve` | `PATCH` | **الموافقة** الرسمية على طلب تسجيل مزود الخدمة وتفعيله. |
| `/providers/:id/reject` | `PATCH` | **رفض** طلب التسجيل مع إرسال سبب الرفض للمزود. |
| `/providers/:id` | `PATCH` | تحديث أي بيانات إدارية خاصة بالمزود. |

### د. إدارة الخدمات العامة (System Services)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/services` | `GET` | عرض قائمة خدمات النظام الأساسية (غسيل، صيانة، سطحة، إلخ). |
| `/services` | `POST` | إنشاء خدمة نظام جديدة وتحديد أيقونتها وسعرها الأساسي. |
| `/services/:id` | `PATCH` | تعديل بيانات الخدمة (الاسم، الوصف، الترتيب). |
| `/services/:id` | `DELETE` | حذف خدمة من قائمة خدمات النظام. |

### هـ. إدارة باقات الاشتراك (Membership Plans)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/memberships` | `GET` | عرض كافة باقات الاشتراك المتاحة للمستخدمين. |
| `/memberships` | `POST` | إنشاء باقة اشتراك جديدة (السعر، المزايا، الخصومات). |
| `/memberships/:id` | `PATCH` | تعديل تفاصيل باقة اشتراك موجودة. |
| `/memberships/:id` | `DELETE` | إزالة باقة اشتراك من المنصة. |
| `/memberships/subscribers` | `GET` | عرض قائمة بجميع المستخدمين المشتركين حالياً في الباقات. |

### و. إدارة الطلبات والحجوزات (Bookings Management)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/bookings` | `GET` | عرض السجل الكامل للطلبات في المنصة. |
| `/bookings/:id` | `GET` | جلب تفاصيل الطلب (الموقع، العميل، المزود، الفاتورة). |
| `/bookings/:id/status` | `PATCH` | تغيير حالة الطلب يدوياً (تأكيد، إتمام، إلغاء). |
| `/bookings/:id` | `DELETE` | حذف سجل طلب من النظام. |

### ز. الإحصائيات (Stats & Analytics)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/stats` | `GET` | ملخص عام (إجمالي المستخدمين، الأرباح، عدد الطلبات). |
| `/stats/bookings` | `GET` | إحصائيات الطلبات مقسمة حسب النوع والحالة. |
| `/stats/revenue` | `GET` | تقرير الأرباح الشهرية (آخر 12 شهر). |
| `/stats/top-services` | `GET` | عرض أكثر الخدمات طلباً وربحية. |

### ح. الإعدادات العامة (Settings)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/settings` | `GET` | جلب إعدادات التطبيق الحالية (وضع الصيانة، ميزات مفعّلة). |
| `/settings/maintenance` | `PATCH` | **تفعيل/إيقاف وضع الصيانة** مع تحديد رسالة التنبيه. |

### ط. إدارة حسابات الأدمن (Super Admin)
> يتطلب صلاحية `Permissions: all`

| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/list` | `GET` | عرض قائمة بجميع من لديهم صلاحيات أدمن. |
| `/create` | `POST` | إنشاء حساب أدمن جديد وتحديد صلاحيات محددة له. |
| `/:id/permissions` | `PATCH` | تعديل صلاحيات أدمن موجود. |
| `/:id/status` | `PATCH` | تعطيل أو تفعيل حساب أدمن. |
| `/:id` | `DELETE` | حذف حساب أدمن نهائياً. |

## 2. العمليات الإدارية في الأقسام الأخرى

### أ. قسم المستخدمين (Users Module)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/users` | `GET` | عرض كل المستخدمين بنظام التمرير (Admin Only). |
| `/users/:id` | `GET` | جلب بيانات مستخدم بالـ ID (Admin Only). |
| `/users/:id` | `PATCH` | تعديل بيانات مستخدم (Admin Only). |
| `/users/:id` | `DELETE` | حذف مستخدم (Admin Only). |

### ب. قسم مزودي الخدمة (Providers Module)
| المسار (Endpoint) | الطريقة (Method) | الوصف |
| :--- | :--- | :--- |
| `/providers/:id/approve` | `POST` | الموافقة السريعة على مزود (Admin Only). |

---

> **ملاحظة**: جميع طلبات الأدمن يجب أن تُرسل مع `Authorization: Bearer <JWT_TOKEN>` في الـ Header.
